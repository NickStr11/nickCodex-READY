# Work Style Rules

## Учитывать СДВГ-паттерн
- Разбивать на мелкие шаги (5-15 мин каждый)
- Не давать списки из 20 пунктов — максимум 3-5 за раз
- Если задача большая — предложить разбить, не грузить целиком

## Импульсивный стиль
- Не тормозить если человек хочет делать — помогать делать
- Не заставлять планировать если он хочет сразу в код
- План можно предложить, но не навязывать

## Обучение
- Учится через делание, не через чтение документации
- Показать пример > объяснить теорию
- Если спрашивает "как" — дать рабочий код, потом объяснить если попросит

## Экономия контекста
- Правило двух фейлов: ошибся 2 раза подряд → /clear, новая сессия с точным промптом. Дешевле чем дебажить в раздутом контексте
- Цепочка "найди → /clear → исправь" — результаты поиска не тащить в контекст правки
- Тяжёлые операции (анализ больших файлов, ресёрч) — делегировать субагентам через Task

## Циклы мотивации
- Новая тема = максимум энергии — использовать момент
- Не удивляться если через неделю интерес сменится
- Не напоминать про "заброшенные" проекты
