# Learned The Hard Way

Grep-able слой для повторяющихся практических уроков.

Это не diary и не правила поведения. Формат нужен, чтобы перед фиксом быстро
найти похожий симптом и не решать одну и ту же проблему заново.

## Как использовать

- Если есть production-баг, странный симптом, повторяющийся сбой или déjà vu —
  сначала искать здесь по симптому.
- Если после фикса стало понятно “симптом → причина → фикс”, добавить короткую
  запись.
- Не превращать файл в дневник. Один урок = один компактный рецепт.

## Формат

```markdown
### <Симптом>
**Причина:** <что реально происходило>
**Фикс:** `<команда или действие>`
**Источник:** <diary/session/research/date>
```

## Recipes

### Агент уверенно пишет числа без проверки
**Причина:** researcher/recon может смешать README, чужой отчёт и догадку в один уверенный вывод.
**Фикс:** применять `rules/research-verification.md`; числа проверять через `gh api`, `git`, `curl` или прямое чтение source file.
**Источник:** Cortex infra report 2026-05-01; Hermes Agent stars failure.

### Хочется сразу сделать SQLite/FTS индекс для маленького корпуса
**Причина:** архитектурное решение появляется раньше замера; raw `rg`/grep может быть быстрее индекса на десятках файлов.
**Фикс:** сначала замерить raw search; FTS/индекс добавлять только когда корпус и latency реально требуют.
**Источник:** Cortex infra report 2026-05-01; session-search FTS5 откат.

### Полезный урок теряется в diary
**Причина:** diary хранит историю сессии, но плохо работает как lookup по симптому.
**Фикс:** переносить стабильный рецепт в этот файл в формате “симптом → причина → фикс → источник”.
**Источник:** Cortex learned-the-hard-way pattern, 2026-05-01.
