# Daily Session

## Workflow

1. Read `AGENTS.md`, then `memory/PROJECT_CONTEXT.md`, `memory/DEV_CONTEXT.md`, `inbox/now.md`, and `inbox/backlog.md`.
2. If the live task points to a specific subtree, read the nearest local `AGENTS.md` there before editing.
3. Pick exactly one active track. If several candidates compete, choose by immediacy, leverage, and user pull.
4. Rewrite the chosen track into a small session scope: one focus line and 1-3 concrete next actions.
5. Update `inbox/now.md` so it matches reality right now, not a wishlist.
6. Leave non-current items in `inbox/backlog.md` or move them into `memory/` or `knowledge/` if they became durable context.
7. Start doing the work. Do not write a long plan unless the user explicitly asks for one.

## Output Rules

- Keep `inbox/now.md` readable in under a minute.
- Keep one active focus only.
- Prefer verbs and concrete next actions over vague goals.
- If the user's latest request already defines the focus, align `inbox/now.md` to it instead of inventing a parallel track.
