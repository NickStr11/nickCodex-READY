# Runtime

Рабочий слой для сырья и временных файлов.

Структура:
- `imports/` — входящие сырые данные и захваченные артефакты
- `outputs/` — результаты генерации и выгрузки
- `research/` — временные ресёрч-материалы
- `scratch/` — черновики и одноразовые файлы

Если файл нужен только на время работы, его место здесь, а не в корне репозитория.
