# Reflections

Промежуточный слой для цикла `diary -> reflection -> rules`.

Что сюда писать:
- короткий синтез повторяющихся паттернов из `memory/diary/` и `memory/sessions/`
- какие правила, skills или memory-файлы были обновлены
- что сознательно не стали превращать в правило

Что держать рядом:
- `processed.log` — список diary/session файлов, которые уже были обработаны reflection-проходом

Не используй эту папку как второй `memory/DEV_CONTEXT.md`.
Если вывод стал устойчивым правилом, перенеси его в `rules/`, `skills/`, `knowledge/` или другой узкий owner.
