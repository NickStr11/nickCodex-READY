# Diary

Журнал завершённых сессий.

Что сюда писать:
- что реально было сделано
- какие решения приняты
- где остановились
- с каким конкретным следующим входом можно продолжить

Формат имени:
- `YYYY-MM-DD-short-slug` как stem имени, сама запись хранится как Markdown note

Держи записи короткими:
- не сырой чат
- не длинная теория
- не второй `memory/DEV_CONTEXT.md`

Использование:
- `inbox/now.md` — текущее состояние и следующий шаг
- `memory/diary/` — история сессий
- `memory/DEV_CONTEXT.md` — временный переходный слой, пока diary-flow не устоится
